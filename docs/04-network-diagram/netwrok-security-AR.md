## 1. نظرة عامة

تصف هذه الوثيقة البنية التحتية لشبكة بنك النور للتمويل والاستثمار، وتشمل التصميم المنطقي للشبكة، وتقسيم الشبكات الافتراضية (VLANs)، وآليات الحماية المطبقة وفق معيار ISO 27001.

تتكون الشبكة من ثلاثة مواقع:
- **المركز الرئيسي** في السويداء
- **الفرع الأول**
- **الفرع الثاني**

يرتبط كل فرع بالمركز الرئيسي عبر نفق VPN Site-to-Site مشفر.

---

## 2. التصميم المنطقي

تعتمد الشبكة على ثلاث مناطق أمنية منفصلة:

| المنطقة | الوصف | مستوى الأمان |
| --- | --- | --- |
| **DMZ** | تستضيف الخدمات المكشوفة للإنترنت (Web Server, ATM GW) | متوسط (50) |
| **Internal Network** | شبكة الموظفين الداخلية مقسمة بـ VLANs | عالي (100) |
| **Secure Zone** | السيرفرات الحساسة وقواعد البيانات | عالي جداً |

## 3. الأجهزة المستخدمة

| الجهاز | الموديل | الدور |

## 6. الخدمات المطبقة

### 6.1 DHCP
- سيرفر DHCP مركزي في الـ Secure Zone (192.168.80.10)
- يتم توجيه طلبات الـ DHCP عبر `ip helper-address` على كل SVI في MS1
- كل VLAN عنده Pool منفصل

### 6.2 OSPF
- بروتوكول التوجيه الديناميكي المستخدم بين MS1 وMS2
- يضمن التحويل التلقائي للمسارات عند فشل أي رابط
- MS1: Router-ID 1.1.1.1 | MS2: Router-ID 2.2.2.2

### 6.3 VTP
- MS1 يعمل كـ VTP Server
- جميع الـ Distribution Switches تعمل كـ VTP Clients
- Domain: BankNoor

### 6.4 SSH
- تم تفعيل SSH v2 على جميع الأجهزة
- تم تعطيل Telnet لمنع نقل البيانات بنص واضح

## 7. سياسة الوصول (ACLs)

تم تطبيق ACLs موسعة على الـ SVIs في MS1 للتحكم في الوصول بين الـ VLANs والـ Secure Zone:

| الـ VLAN | السياسة |
| --- | --- |
| IT (10) | وصول كامل لكل السيرفرات |
| Security (20) | وصول كامل لكل السيرفرات |
| Finance (30) | وصول لـ DB-Server وHR-Server فقط |
| HR (40) | وصول لـ HR-Server فقط |
| CustomerService (50) | وصول لـ DB-Server فقط |
| Management (60) | محجوب كلياً من الـ Secure Zone |
| Reception (70) | محجوب كلياً من الـ Secure Zone |

## 8. الـ Firewall (ASA 5506-X)

| الـ Interface | الاسم | الـ IP | Security Level |
| --- | --- | --- | --- |
| Gig1/1 | inside | 10.0.0.1 | 100 |
| Gig1/2 | dmz | 192.168.90.1 | 50 |
| Gig1/3 | outside | 203.0.113.1 | 0 |

**السياسات المطبقة:**
- الـ DMZ مسموح لها استقبال HTTP/HTTPS من الإنترنت فقط
- الـ DMZ محجوبة من الوصول للـ Secure Zone
- الـ Internal Network مسموح لها الوصول للـ DMZ

> ملاحظة: تم ضبط الـ ASA بشكل كامل مع security levels وACLs. بعض اختبارات ICMP محدودة بسبب قيود محاكي Packet Tracer على موديل ASA 5506-X.

---

## 9. الفروع

يرتبط كل فرع بالمركز الرئيسي عبر **VPN Site-to-Site** مشفر. كل فرع يحتوي على:
- Router يدير نفق الـ VPN
- Switch للتوزيع الداخلي
- أجهزة الموظفين

تنطبق على الفروع نفس سياسات الـ VLANs والـ ACLs المطبقة في المركز الرئيسي.

---

## 10. نقاط القوة والتحسينات المقترحة

**نقاط القوة:**
- تقسيم الشبكة لمناطق أمنية واضحة
- ACLs دقيقة تطبق مبدأ Least Privilege
- DHCP مركزي في الـ Secure Zone
- Redundancy عبر MS1 وMS2

**تحسينات مقترحة مستقبلاً:**
- إضافة IDS/IPS مستقل
- تطبيق 802.1X للتحقق من هوية الأجهزة قبل الاتصال بالشبكة
- تشفير قواعد البيانات
