## 1. الهدف

توفير إطار أمني شامل لحماية البنية التحتية للشبكة (السلكية واللاسلكية) وضمان سرية البيانات أثناء انتقالها، ومنع الوصول غير المصرح به إلى موارد الشبكة الداخلية.

---

## 2. النطاق

تشمل هذه السياسة جميع مكونات الشبكة التابعة للمنظمة، بما في ذلك:

- أجهزة الشبكة (الموجهات، المحولات، جدران الحماية).
- الشبكات اللاسلكية (Wi-Fi).
- اتصالات الطرف الثالث والوصول عن بُعد (VPN).
- نقاط الاتصال بالإنترنت.

---

 ## 3. البنود والقواعد (Policy Provisions)

### - **تقسيم الشبكة (Network Segmentation):** 
يجب فصل الشبكة إلى مناطق أمنية معزولة (مثل عزل شبكة الضيوف عن شبكة الإدارة، وعزل بيئة التطوير عن بيئة الإنتاج).
### - **جدران الحماية (Firewalls):** 
يجب تفعيل جدران حماية عند جميع نقاط الاتصال بالإنترنت، مع تطبيق قاعدة "الرفض الضمني" (Deny by Default) والسماح فقط بحركة المرور الضرورية.
### - **أمن الشبكة اللاسلكية:** 
يُمنع استخدام بروتوكولات التشفير الضعيفة (مثل WEP أو WPA). يجب استخدام (WPA3 أو WPA2 Enterprise) مع مصادقة قوية.
### - **تشفير البيانات أثناء النقل:** 
يجب تشفير جميع البيانات الحساسة المتبادلة عبر الشبكة باستخدام بروتوكولات آمنة (مثل TLS/SSL أو SSH).
### - **كشف ومنع التسلل (IDS/IPS):** 
يجب نشر أنظمة لمراقبة حركة مرور الشبكة واكتشاف الأنشطة المشبوهة أو الهجمات السيبرانية والتصدي لها تلقائياً.

## 4. المسؤوليات

| الجهة           | المسؤولية                                                                                                     |
| --------------- | ------------------------------------------------------------------------------------------------------------- |
| مهندسو الشبكات  | مسؤولون عن الإدارة الفنية للأجهزة، وتحديث برمجياتها (Firmware)، وضبط إعدادات الأمان.                          |
| الأمن السيبراني | مسؤول عن مراقبة سجلات الشبكة (Logs)، وإجراء اختبارات الاختراق الدورية، والتأكد من فعالية سياسات جدار الحماية. |
| المستخدمون      | الالتزام بعدم توصيل أي أجهزة غير مصرح بها (مثل أجهزة الراوتر الشخصية) بالشبكة الداخلية.                       |
|                 |                                                                                                               |
|                 |                                                                                                               |

## 5. العقوبات

- يُعد التلاعب بإعدادات الشبكة، أو محاولة كسر التشفير، أو القيام بمسح للثغرات (Scanning) دون تصريح، مخالفة أمنية جسيمة.
- تتراوح العقوبات بين الإنذار النهائي والفصل من الخدمة، مع تحمل المسؤولية القانونية في حال التسبب في تعطل الشبكة أو تسريب بياناتها.
---

## 6. المراجعة

تُراجع قواعد جدار الحماية (Firewall Rules) كل **6 أشهر**، وتُحدث السياسة سنوياً لمواكبة التطورات في تقنيات الاتصال والتهديدات المرتبطة بها.
