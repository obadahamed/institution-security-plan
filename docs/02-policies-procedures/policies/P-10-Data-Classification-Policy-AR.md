## 1. الهدف

الهدف من هذه السياسة هو إنشاء إطار عمل لتصنيف بيانات المنظمة إلى فئات محددة، مما يضمن تخصيص مستويات الحماية المناسبة لكل فئة بناءً على مستوى حساسيتها والمخاطر المترتبة على فقدانها أو تسريبها.

---

## 2. النطاق
تنطبق هذه السياسة على جميع البيانات والمعلومات التي تمتلكها أو تعالجها المنظمة، سواء كانت في شكل **رقمي** (قواعد بيانات، ملفات إلكترونية) أو **مادي** (وثائق ورقية، تقارير مطبوعة).

---

 ## 3. مستويات التصنيف (Classification Levels)

يتم تصنيف البيانات إلى أربعة مستويات رئيسية:

- **سري للغاية (Confidential/Secret):** البيانات الأكثر حساسية (مثل الأسرار التجارية، استراتيجيات المنظمة، بيانات العملاء البنكية). تسريبها يسبب ضرراً كارثياً.
- **داخلي (Internal Use):** بيانات مخصصة للاستخدام داخل المنظمة فقط (مثل الهياكل التنظيمية، السياسات الداخلية، المذكرات الرسمية). تسريبها يسبب إحراجاً أو ضرراً تشغيلياً بسيطاً.
- **مقيد (Restricted):** بيانات يحق لفئة معينة فقط الاطلاع عليها (مثل بيانات رواتب الموظفين، تقارير الأداء).
- **عام (Public):** البيانات المخصصة للنشر (مثل المنشورات التسويقية، التقارير السنوية المنشورة، أخبار المنظمة).
---
## 4. بنود التعامل مع البيانات (Handling Rules)

### - **التوسيم (Labeling):** يجب وضع علامة واضحة على الوثائق والملفات توضح مستوى تصنيفها (مثلاً: "سري" في رأس الصفحة).
### - **التخزين:** البيانات "السرية للغاية" يجب أن تُخزن مشفرة مع تقييد الوصول إليها بمصادقة MFA.
### - **التخلص من البيانات:** يجب إتلاف الوثائق الورقية "السرية" باستخدام آلات تمزيق الورق، ومسح البيانات الرقمية باستخدام برامج تمنع استعادتها.
### - **النقل:** يُمنع إرسال البيانات "السرية" عبر البريد الإلكتروني الشخصي أو وسائط التخزين غير المشفرة.

---


## 4. المسؤوليات

| الجهة                      | المسؤولية                                                                                                                            |
| -------------------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| مالك البيانات (Data Owner) | هو المدير المسؤول عن القسم الذي أنشأ البيانات، وهو المسؤول عن تحديد مستوى تصنيفها ومراجعة صلاحيات الوصول إليها دورياً.               |
| أمين البيانات              | عادة ما يكون فريق تقنية المعلومات، وهو المسؤول عن تنفيذ الوسائل التقنية لحماية البيانات (تشفير، نسخ احتياطي) بناءً على تصنيف المالك. |
| المستخدمون                 | مسؤولون عن اتباع قواعد المناولة والتخزين الخاصة بكل تصنيف.                                                                           |
|                            |                                                                                                                                      |
|                            |                                                                                                                                      |

---

## 5. العقوبات

- يُعد سوء تصنيف البيانات أو معالجة البيانات الحساسة بطريقة تخالف المعايير الأمنية (مثل ترك وثائق سرية في مكان عام) مخالفة تستوجب التنبيه أو الإنذار.
- تسريب البيانات المصنفة "سرية للغاية" عمداً أو نتيجة إهمال جسيم يؤدي إلى إنهاء الخدمة والملاحقة القانونية.

---

## 6. المراجعة

تُراجع مستويات تصنيف البيانات بشكل سنوي، أو عند تغير طبيعة النشاط التجاري للمنظمة لضمان دقة التصنيفات الحالية.
