## 1. الهدف

حماية البنية التحتية للشبكة (السلكية واللاسلكية) وضمان سرية البيانات أثناء انتقالها، ومنع الوصول غير المصرح به إلى موارد الشبكة الداخلية.

---

## 2. النطاق

جميع مكونات الشبكة التابعة للمنظمة، بما في ذلك:

- أجهزة الشبكة (الموجهات، المحولات، جدران الحماية).
- الشبكات اللاسلكية (Wi-Fi).
- اتصالات الطرف الثالث والوصول عن بُعد (VPN).
- نقاط الاتصال بالإنترنت.

---

## 3. الخطوات التفصيلية

### الخطوة 1 — إجراء عزل وتقسيم الشبكة (Segmentation):


- تقسيم الشبكة إلى مناطق (VLANs) لعزل حركة مرور بيانات الإدارة عن بيانات الموظفين، وعزل شبكة الضيوف (Guest Wi-Fi) تماماً.

- وضع الأنظمة الحساسة (مثل السيرفرات) في منطقة محمية خلف جدار حماية (DMZ).
### الخطوة 2 — إجراء إدارة جدران الحماية (Firewall Management):

- تطبيق قاعدة "المنع الضمني" (Deny All) والسماح فقط بالخدمات الضرورية (مثل البريد أو التصفح الآمن).

- مراجعة قواعد جدار الحماية كل 6 أشهر لحذف القواعد القديمة أو غير المستخدمة.

### الخطوة 3 —إجراء تأمين الاتصالات اللاسلكية (Wi-Fi Security):

- استخدام بروتوكولات تشفير قوية (مثل WPA3 أو WPA2-Enterprise) مع منع استخدام البروتوكولات القديمة (WEP).

- إخفاء اسم الشبكة (SSID) للأنظمة الحساسة وتغيير كلمات مرور الواي فاي بشكل دوري.
### الخطوة 4 — إجراء تشفير البيانات أثناء النقل (Encryption in Transit):

- إلزامية استخدام بروتوكولات مشفرة لجميع العمليات (مثل HTTPS للمواقع، و SSH للإدارة، و TLS للبريد).
### الخطوة 5 — إجراء كشف ومنع التسلل (IDS/IPS):

- تفعيل أنظمة مراقبة حية تقوم بتحليل حزم البيانات واكتشاف الأنماط المشبوهة (مثل محاولات الاختراق) وحظرها تلقائياً.

## 4. المسؤوليات

| الجهة                     | المسؤولية                                                                                             |
| ------------------------- | ----------------------------------------------------------------------------------------------------- |
| **الموظفون**              | يُمنع منعاً باتاً توصيل أجهزة توجيه (Routers) شخصية أو أجهزة "نقطة وصول" غير معتمدة بالشبكة الداخلية. |
| **فريق الأمن السيبراني:** | مراجعة سجلات الشبكة (Logs) لاكتشاف الثغرات، وإجراء اختبارات الاختراق (Penetration Testing) الدورية.   |
| **مهندس الشبكات**         | مسؤول عن الإعدادات الفنية، تحديث البرامج الثابتة (Firmware) للأجهزة، وضمان استقرار الاتصال            |

## 6. المراجعة

- يُراجع هذا الإجراء **سنوياً** أو فور وقوع أي حادثة أمنية مرتبطة بالخطأ البشري.
- أي تعديل يستوجب موافقة مدير أمن المعلومات وتوثيقه في سجل التغييرات.
